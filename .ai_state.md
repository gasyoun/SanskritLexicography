## 🧠 Dev Notes & Hypotheses

- 2026-06-16: **PLACEHOLDER-FABRICATION BUG caught by a live integrity audit —
  the single most important corpus lesson.** A 5-auditor + adversarial-verify
  workflow over the live `corpus_lexicon.jsonl` found that works whose corpus
  `seg=ru` is the placeholder `…` (no Cyrillic) — `12_mahabharata-shantiparva`
  (99.8% placeholder), `13_…-anushasanaparva` (100%) — were still aligned, and
  DeepSeek **hallucinated** fluent Russian for translations that do not exist:
  **166k of 204k rows (81%) were invention** attributed to real verse citations.
  They pass every surface check (fluent Cyrillic) → invisible to QA. **Lesson:
  truthiness ≠ translated; a real ru MUST contain Cyrillic.** Fix = `has_cyr()`
  guard in `pairs_of()` (placeholder → 0 API calls) + write-step Cyrillic/`ru!=sa`/
  refusal guards + `SYS_BATCH` never-invent + `[NOTE]` commentary instruction +
  `√`-strip + per-batch `(group,slp1,ru,kind)` dedup. `build_strata.count_groups`
  now requires Cyrillic ru (total 78,139→58,897). Recovery via `_purge_lexicon.py`
  → 26,277 genuine rows (0 untranslated-group / 0 non-Cyrillic / 0 `ru==sa` / 0
  dup, verified). Committed + pushed; build restarted clean. **Always re-scan a
  generative corpus build for source-placeholder leakage before trusting volume.**
- 2026-06-16: **Batching verses/call holds quality.** A/B (5 Gītā verses, batched
  1-call vs 5 single-calls) = identical Russian glosses; batched returns cleaner
  dictionary-form keys (`samaveta` vs inflected `samavetā`) and keeps verses
  independent (no word-bleed). ~3-4× faster wall-clock, cost ~unchanged
  (content-bound). `_watch.py` rewritten to measure a live window (the append-only
  lexicon's ctime is stale across runs → bogus 24-min/$169 ETA).
