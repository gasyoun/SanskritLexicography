# Как стать соратником проекта — путь нового участника

_Created: 16-07-2026 · Last updated: 16-07-2026_

Это руководство — для человека, который **знает санскрит (пусть на начальном
уровне)** и хочет помогать проекту, но пока **не умеет работать ни с Claude, ни
с git, ни с GitHub**. Ничего страшного: инструменты осваиваются по ходу, на
реальной пользе. Твоя санскритология здесь ценнее любого программирования —
человек, читающий деванагари, видит ошибку там, где машина её не замечает.

Более «взрослые» документы — английский
[CONTRIBUTING.md](https://github.com/gasyoun/SanskritLexicography/blob/master/CONTRIBUTING.md)
(предполагает знание git) и русский обзорный
[MANUAL_LEXICON_WORKSPACE_HUMAN_RU.md](https://github.com/gasyoun/SanskritLexicography/blob/master/MANUAL_LEXICON_WORKSPACE_HUMAN_RU.md)
(для тех, кто уже глубоко в проекте). Их читать **не сейчас** — вернёшься к ним
на уровне 2–3.

## Что это за проект (в двух словах)

Мы оцифровываем и вычитываем санскритские словари (Monier-Williams, Apte,
Böhtlingk и др.) — это Cologne Digital Sanskrit Dictionaries. Тексты уже
отсканированы и распознаны, но в них остаются ошибки распознавания (OCR),
разметки и ссылок. Наша задача — находить эти ошибки и аккуратно их
исправлять.

**Важное правило с самого начала:** мы **никогда не правим исходный файл
словаря напрямую.** Найденная ошибка оформляется как «issue» (заметка-задача)
на GitHub; правки вносятся потом, пакетом, через проверку. От новичка нужно
только одно — **находить ошибки и понятно их описывать.**

Claude (ИИ-помощник) нужен не «вместо тебя», а как напарник: он оформит
найденное правильно, объяснит незнакомое, подскажет команды. С кода начинать
не надо совсем.

## Лестница: от пользователя к полноценному соратнику

Идти по ступеням сверху вниз. Каждая — это новое умение, а не экзамен. На
любой можно задержаться сколько нужно.

| Ступень | Что осваиваешь | Признак, что готов дальше |
|---|---|---|
| **0. Разговор с Claude** | Просто общаться с ИИ по-русски | Свободно задаёшь вопросы и поправляешь его |
| **1. GitHub + первая польза** | Аккаунт, интерфейс, оформить ошибку как issue | Создан(ы) первый(е) issue (см. «Первое задание») |
| **2. Правки через сайт** | Fork, ветка, pull request прямо в браузере GitHub | Слит первый маленький PR (правка текста/документа) |
| **3. Claude Code на компьютере** | Установить программу, запустить её в папке репозитория, сделать задание по инструкции (handoff) | Выполнен один guided-handoff под присмотром |
| **4. Самостоятельный контрибьютор** | Брать задачи из очереди, вести их до конца сам | Регулярно закрываешь задачи без пошаговой опеки |

Цель — довести тебя до ступени 4. Но польза проекту начинается уже на ступени 1.

## Ступень 0 — доступы и первое знакомство (полчаса + пара вечеров)

1. **Аккаунт GitHub** — [github.com/signup](https://github.com/signup). Пришли
   свой ник тому, кто тебя пригласил, — тебя добавят в организацию
   [sanskrit-lexicon](https://github.com/sanskrit-lexicon).
2. **Claude** — [claude.ai](https://claude.ai), вход через Google/почту. Пока
   этого достаточно; **ничего устанавливать не надо.** Программу Claude Code
   поставим на ступени 3.
3. **Освойся с Claude без задания** — просто поговори с ним по-русски о том,
   что знаешь:
   - «Объясни разницу между значениями корня √dhā в словаре Monier-Williams.»
   - «Вот статья из словаря *(вставь текст)* — есть ли в ней опечатки в
     транслитерации?»

   Цель — почувствовать, что с ним можно разговаривать обычными словами,
   просить переспросить, поправлять его. «Правильного синтаксиса» нет.

## Ступень 1 — первое задание (ограниченное, проверяемое, полезное)

**Важно: это не «читать словарь подряд».** Такое блуждание бесполезно —
непонятно, что искать и когда остановиться. Задание другое и с ясным концом:
**машина уже нашла подозрительные слова и свела их в список. Не хватает
человека, который откроет фотографию печатной страницы и скажет: машина права
(в оцифровке ошибка) или машина ошиблась (в книге напечатано так).** Каждый
ответ — прямая польза: именно этот список держит словари перед печатью
(«ворота A», см.
[PRINT_READINESS.md](https://github.com/gasyoun/SanskritLexicography/blob/master/HeadwordLists/PRINT_READINESS.md);
весь спайн MW+PWG держат всего 16 таких слов).

**🎯 Задание: сверь по сканам 3–4 слова из списка подозрительных.**

1. Открой рабочий список:
   [A_TYPO_QUEUE.md](https://github.com/gasyoun/SanskritLexicography/blob/master/HeadwordLists/A_TYPO_QUEUE.md).
   Найди строки, помеченные словарём **MW** (их всего 4 — начни с них). В каждой
   строке: как слово сейчас в оцифровке, как — по мнению программы — должно
   быть, и тип подозрения («долгота a↔ā», «шипящий s→ś», «церебральный n→ṇ» и
   т. п.).
2. Для каждого слова открой его в **Monier-Williams (MW)** на Кёльнском сайте
   ([sanskrit-lexicon.uni-koeln.de](https://www.sanskrit-lexicon.uni-koeln.de/)).
   Рядом со статьёй есть ссылка на **скан** печатной страницы — посмотри именно
   на это место в книге.
3. Реши, права ли машина, и запиши по шаблону:

   > **Слово:** bhaṭṭāraka (MW)
   > **Машина подозревает:** должно быть `bhaṭṭāraka` (с долгим ā), сейчас в
   > оцифровке `bhaṭṭaraka`
   > **По скану:** подтверждаю — в книге ā _(или)_ опровергаю — в книге короткое a
   > **Ссылка на скан:** …

4. **Оформить это в issue поможет Claude.** Скажи ему по-русски: *«Помоги мне
   оформить для GitHub результат сверки словарного слова, вот что я проверила:
   …»* — и приложи свои заметки. Готовые issue создаются на репозитории словаря
   ([github.com/sanskrit-lexicon](https://github.com/sanskrit-lexicon)); не
   знаешь, куда нажать, — пришли текст тому, кто тебя ведёт.

**Задание выполнено, когда** по 3–4 словам есть чёткий вердикт
«подтверждаю / опровергаю» и ссылка на скан. **Оба ответа ценны:**
подтверждение → правку внесут в словарь; опровержение → снимут ложное
срабатывание машины. Проигрышного варианта нет.

**Не переживай про** ярлыки, приоритеты, git, «правильный» формат issue — это
надстройка, её добавим вместе позже.

### Вариант Б — если санскрит совсем начальный

Четыре MW-слова различаются тонкими значками (ś / ṣ / s, ṇ / n) — если
деванагари пока даётся тяжело, возьми задание, где санскрит не нужен вовсе:
**вычитай английскую часть одной страницы словаря.** Тоже ограниченно (одна
страница/колонка, не «весь словарь») и проверяемо.

1. Выбери в **Apte (AP90)** — там английского текста больше всего — одну
   колонку скана и прочитай подряд её английские определения, сверяя со сканом.
2. Отмечай: опечатки в английских словах; искажённые латинские названия
   растений и животных (например, «Pistia Stratiotes»); слипшиеся/разорванные
   слова, потерянный курсив; битые сокращения источников (`L.`, `R.`, `Suśr.`).
3. Оформляется так же — заметка на каждую находку + Claude помогает довести до
   issue.

**Самый простой запасной вариант.** Если и это тяжело — просто отмечай места с
**нечитаемым/битым сканом** или с **мёртвой ссылкой** на другую статью: вопрос
только «читается / не читается» и «ссылка ведёт куда сказано / нет», разбирать
текст не надо. Тоже реальная и нужная работа.

## Ступени 2–4 — куда расти дальше

Подробные шаги дам отдельно, когда дойдёшь; здесь — карта, чтобы видеть
горизонт.

- **Ступень 2 — правки через сайт.** GitHub позволяет делать простые правки
  прямо в браузере (кнопка ✏️ на файле → «предложить изменение» → pull
  request). На этой ступени берёшь мелкую задачу: поправить опечатку в
  документе, дополнить список сокращений, исправить битую ссылку. Claude
  объяснит, что такое fork, ветка и PR, простыми словами.
- **Ступень 3 — Claude Code на компьютере.** Устанавливаешь программу Claude
  Code (командная строка), запускаешь её прямо в папке репозитория. Первое
  задание здесь — «handoff» (`H###`): готовый файл-инструкция, который просто
  открываешь и выполняешь под присмотром. Компьютер у тебя есть — эта ступень
  открыта.
- **Ступень 4 — самостоятельность.** Берёшь задачи из общей очереди
  (`what's next`), ведёшь их до конца, оформляешь результат по нашим
  соглашениям. С этого момента ты — полноценный соратник, а не пользователь
  данных.
