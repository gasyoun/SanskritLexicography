**Mechanism (new, distinct from the already-fixed H960 class).** All 8 are **Nachtrag/corrigenda** cards: the PWG source bundles textual-correction points for **several different compound-verb sub-lemmas** (each opened by its own `— {#XX#}` marker) under one addendum entry, and the pipeline stores/translates the whole bundle as **one** sense. `count_source_senses` correctly finds each sub-block's own line-opening ordinal — by design, H960's cross-reference hardening only excludes a **mid-prose** ordinal (e.g. `vgl. u. 2. {#mArga#} 2)`), not a legitimately line-opening ordinal that happens to belong to a *different* sub-lemma than the one the row is filed under. The counter has no notion of "this ordinal belongs to a different stored unit" — so the distinct-ordinal count across the bundle exceeds the stored-sense count even though **nothing is missing**. Example (`y_a~~h5_00_pwg00`, the most extreme case): one stored sense bundles corrigenda points for `{#yA#}` (10), `{#samA#}` (3), `{#nis#}` (1), and `{#vi#}` (2) — all four points' text is verbatim present and translated in both `de` and `ru`.

This is confirmed by direct inspection of the RU field, not assumed from the German echo alone — e.g. `n_i~~h5_10_pari`'s `3-corr-8` sense translates literally as `вместо 8) следует читать` ("instead [read] 8) ...") — the "8)" content the guard treats as a missing separate sense is right there, translated.

This is a **different, currently-unfixed** class from the three H960-fixed cross-reference examples already on record (`gam~~h2_31_pari` 2→1, `s_ud~~h0_05_pra` 4→2, `_a_srayatva` 2→0, per [ARCHITECTURE §3.2](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/docs/ARCHITECTURE_SanskritLexicography_PWG_RU_UNFREEZE.md)) — this measurement confirms none of those three are flagged today (the H960 fix holds), while surfacing a distinct bundling-across-sub-lemmas mechanism H960 never targeted.

**SANLOSS recommendation: `FIX_COUNTER_FIRST`.** Every flag in the available frozen evidence is a false flag (8/8) driven by one identifiable, describable, and — per the fix suggestion below — plausibly fixable mechanism. Arming `SANLOSS_HARD_REJECT` today would requeue-reject already-complete, already-faithfully-translated Nachtrag cards for zero real quality benefit, burning agent budget the lane is already D-3-blocked on. Fix suggestion: partition `count_source_senses`'s ordinal search by `— {#headword#}` sub-lemma boundary before stamping `source_senses`, or (cheaper) have `accept()` check whether an "extra" ordinal's content is verbatim present somewhere in the emitted card before counting it as a shortfall. Either closes this class without weakening the existing H960 mid-prose hardening.

### Reconstruction-boundary caveat (read before citing "865" or "16" elsewhere)

This measurement's `raw` (used to compute `ls`/`sk`/`exp`) is a **reconstruction** — the concatenation of a card's own stored senses' `de` fields, in store order — not the true original undivided PWG source text (that text is not persisted anywhere for these already-promoted cards; only the post-split, post-generation store rows survive). Concatenation is a close approximation for `count_source_senses` (each stored sense retains its own line structure, and cross-reference mid-prose ordinals stay correctly excluded), but the 16-of-881 fidelity-gate rejections found under this reconstruction (`<ls>`/`{#` count mismatches between the concatenated German echo and the RU translation) are very likely a symptom of the true generation-time card boundary differing from my subcard-grouping reconstruction in a handful of cases, or of the promoted store predating the H1152 translation-fidelity guard-2 fix — **not** a live, currently-active production defect (guard-2's `<ls>`/`{#` hard check is unconditional and already active; a residual mismatch surviving in the store most plausibly predates that check). This is out of this handoff's scope (SANLOSS/TNMASK soft-guard arming) and is reported here for transparency, not investigated further.

**Structural blind spot (read before treating "0 true drops" as proof the guard never has real hits).** This reconstruction can only ever find a false-flag-shaped mismatch: a genuinely *dropped* sense's text is, by definition, absent from the promoted store, so it can never inflate the recomputed `exp` above what remains visible. The 8/8 false-flag result says the *frozen, already-promoted* evidence contains no case where the guard fires for a reason other than sub-lemma bundling — it does **not** independently prove the guard would never correctly catch a genuine drop (H994's synthetic canary, cited below, already proves that side of the guard mechanically); it only bounds the false-flag side, on this route, under this payload regime.
