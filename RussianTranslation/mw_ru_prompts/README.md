# Промпты перевода `mw_ru` — по одному файлу на этап

Системные промпты каждого этапа сборки русского перевода Monier-Williams
(`mw_ru`). Извлечены из пофайловых логов прогонов (`_meta.system_prompt` /
`system_blocks`). Контекст процесса — [../mw_ru.md](../mw_ru.md).

| Файл | Этап | Модель |
|------|------|--------|
| [1_perevod.txt](1_perevod.txt) | этап 1 — основной перевод (+ глоссарий 91 пара) | Claude Sonnet 4.6 (и др.) |
| [2_qa_sudya_opus.txt](2_qa_sudya_opus.txt) | этап 2 — QA-аудит, судья 1 (инструкция + калибровка) | Claude Opus 4.7 |
| [2_qa_sudya_yandexgpt.txt](2_qa_sudya_yandexgpt.txt) | этап 2 — QA-аудит, судья 2 | YandexGPT 5.1 |
| [3_pereperevod_opus.txt](3_pereperevod_opus.txt) | этап 3 — переперевод брака (+ глоссарий 145 пар) | Claude Opus 4.7 |

Этап 2 (QA-аудит) — **два промпта**: словарь судили независимо два судьи
по одной рубрике (Opus и YandexGPT), поэтому оба файла под префиксом `2_`.

Этап механических regex-правок промпта не имел (к модели не обращались).
Gemini 2.5 Pro пробовали как судью по той же рубрике, что в этапе 2 —
отдельного промпта нет, эксперимент отвергнут.

Полный запрос к модели = системный промпт (эти файлы) + per-card
user-сообщение (текст карточки).

## Дополнительно

[qa_judge_v4.md](qa_judge_v4.md) — **предложенный** апдейт QA-судьи (v4) к
этапу 2, не применявшийся в текущих прогонах: pre-filter тривиально-OK
карточек до отправки в QA, отдельная категория `tag_dropped`, slim-вариант
промпта для Yandex. Это план обновления судьи для будущих прогонов, а не
системный промпт отдельного этапа сборки `mw_ru`.
