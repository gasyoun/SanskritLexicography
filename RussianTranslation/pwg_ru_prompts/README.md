# Промпты перевода `pwg_ru` — по одному файлу на этап

Системные промпты каждого этапа **планируемой** сборки русского перевода
Petersburger Wörterbuch (большой Бётлингк–Рот, `pwg_ru`). В отличие от
`mw_ru`, этот набор — это **проект/каркас будущего прогона**: ни одна карточка
ещё не переведена, прогоны не запускались, счётчики и логи не сняты. Все
формулировки этапов — в будущем времени (планируется, будет). Контекст процесса —
[../pwg_ru.md](../pwg_ru.md).

Конвейер повторяет `mw_ru` (Sonnet переводит → Opus + YandexGPT судят → Opus
переперевод брака) и добавляет **новый этап 4** — санскритско-русскую
корпусную проверку (**не блокирующий аннотатор**, см. ниже). Метод перевода:
прямой немецко-русский, с опорой (seed) на готовую русскую карточку `mw_ru`
для тех заглавных слов, что есть и в Monier-Williams (как терминологический
ориентир для гармонизации — **не** копирование; переводится именно немецкий
текст PWG).

| Файл | Этап | Модель |
|------|------|--------|
| [1_perevod.txt](1_perevod.txt) | этап 1 — основной перевод нем.→рус. (+ глоссарий нем.→рус.) | Claude Sonnet 4.6 (и др.) |
| [2_qa_sudya_opus.txt](2_qa_sudya_opus.txt) | этап 2 — QA-аудит, судья 1 (инструкция + калибровка) | Claude Opus 4.8 |
| `2_qa_sudya_yandexgpt.txt` (планируется) | этап 2 — QA-аудит, судья 2 | YandexGPT 5.1 |
| [3_pereperevod_opus.txt](3_pereperevod_opus.txt) | этап 3 — переперевод брака (+ расширенный глоссарий) | Claude Opus 4.8 |
| [4_korpus_proverka.txt](4_korpus_proverka.txt) | этап 4 — санскритско-русская корпусная проверка (новый этап) | детерминированный скрипт + индекс С→Р словарей (два сигнала) |

Этап 2 (QA-аудит) — **два судьи**: словарь будут судить независимо два судьи
по одной рубрике (Opus и YandexGPT), поэтому оба файла под префиксом `2_`.
Промпт судьи 2 (`2_qa_sudya_yandexgpt.txt`) ещё не написан — **планируется**;
на диске пока лежит только промпт судьи 1.

Этап 4 — **новый по сравнению с `mw_ru`**: терминальный этап после QA, которого в
`mw_ru` нет. Это не обращение к переводящей модели, а детерминированная сверка
русской заглавной семемы с санскритско-русскими словарями по общему
ключу SLP1 (`<k1>`, с откатом на `<k2>`; нормализация каждого ключа —
длиносохраняющей `form_key()`). По принятому проектному решению этот шлюз —
**не блокирующий аннотатор**: он размечает каждую карточку и **никогда** не
задерживает её. Каждая карточка получает **два независимых сигнала**:

1. **Корректность** — сверка с **независимо составленными** авторитетами:
   согласуется ли русская заглавная семема с независимым санскритско-русским
   словарём? Вердикт: `pass` / `divergence` / `no-check` / `key-mismatch`.
   `divergence` при низкой уверенности уходит в очередь редактора; `no-check`
   (для ключа нет независимой глоссы) пропускается дальше с пометой «не покрыто».
2. **Согласие со справочником (reference-agreement)** — насколько наш ИИ-перевод
   близок к **готовому человеческому** переводу **того же** словаря (KOW,
   PWG→рус.). Градуированная мера: `high` / `partial` / `none` / `no-ref`. Это
   **сходство**, а не проверка истинности.

Почему два сигнала: KOW сам по себе — частичный человеческий перевод PWG→рус.,
посеянный (seed) от Wilson (WIL), поэтому он разделяет родословную с тем текстом,
что мы переводим. Это делает его **идеальным человеческим справочником** (сигнал 2),
но лишь **вторичным**, не решающим подтверждением для **корректности** (сигнал 1):
KOW может поддержать `pass`, но **никогда** не решает `divergence` в одиночку.
Глоссы KOW всегда помечаются как производные от PWG/WIL.

Авторитеты — это **оцифрованные санскритско-русские словари**, поставляемые
мейнтейнером; все принимаются как настоящие русские глоссы. Участники, по роли:

- **Кочергина** — В. А. Кочергина, Санскритско-русский словарь, современный,
  составлен **независимо** → **независимый**, главный якорь сигнала
  корректности; будет предоставлен мейнтейнером.
- **FRI** — словарь Фриша (многоязычный читатель), **уже на диске** в
  `csl-orig/v02/fri/fri.txt`; русская глосса
  лежит в спанах `<lang n="russian">…</lang>` (в файле есть также
  `<lang n="czech">` и `<lang n="english">`, берётся именно русский спан) →
  **независимый**, даёт вклад в корректность уже сейчас.
- **KNA** — санскритско-русский словарь Кнауэра (Knauer) → **независимый**;
  будет предоставлен мейнтейнером.
- **KOW** — Коссович (1854), частичный человеческий перевод PWG→рус.,
  посеянный от WIL → **справочник** для сигнала 2 и **вторичное** подтверждение
  для сигнала 1; мейнтейнер предоставит `kow.txt` (цель —
  `RussianTranslation/src/kow.txt`).
- плюс любые другие оцифрованные С→Р словари, которые добавит мейнтейнер
  (с явным указанием роли — независимый или справочник).

Шлюз соединяет заголовок PWG (`key1`, с откатом на `key2`) с каждым авторитетом,
собирает русскую глоссу(ы) из **всех** доступных С→Р источников, **каждую с
пометой источника и роли** (независимый / справочник), и выносит **раздельные**
вердикты по двум сигналам. **SKD** (Шабдакальпадрума) и **VCP** (Вачаспатья)
остаются санскритско-санскритскими (**ноль** кириллицы) и служат только
санскритской опорой смысла — **никогда** русским авторитетом и **никогда** не
решают корректность; любая конфигурация, отправляющая русскую сверку в SKD/VCP,
обязана жёстко падать.

Охват **не** фиксированный потолок и **не** блокер: он зависит от того, какие
словари загружены, **измеряется при загрузке (ingest)** и честно показывается
**раздельно по сигналам** — охват корректности от Кочергина∪FRI∪KNA, охват
согласия-со-справочником от KOW. FRI даёт охват корректности уже сейчас;
Кочергина, KNA и KOW поднимут его, как только будут поставлены. Точную цифру
охвата заранее не называем — она будет известна, как только источники собраны.
Вырожденный случай: только при **нулевом** числе загруженных С→Р источников этап
вернул бы `CORPUS-UNAVAILABLE`; это не ожидаемое состояние, а граничный.

Этап механических regex-правок промпта иметь не будет (к модели не
обращаются).

Полный запрос к модели = системный промпт (эти файлы) + per-card
user-сообщение (текст карточки).

## Правила перевода (общие для всех этапов)

Переводятся **только** немецкая связная проза и немецкие глоссы `{%…%}`.
Латинские `{%…%}` остаются без изменений. SLP1-санскрит `{#…#}`, ссылки
`<ls>`, сокращения `<ab>`, а также `<is>`/`<lex>`/`<lang>` и структурная
разметка (`<L>`, `<LEND>`, `<div>`, `<H>`, `[PageN-PPPP]`, `¦`, `˚`) —
**неприкосновенны**. Учёная сноска `<F>…</F>` — особый случай: маскируются
**только** сами теги `<F>` и `</F>`, а немецкая проза между ними переводится
как обычная открытая проза (вложенные `{#…#}`/`<ls>` внутри неё при этом
маскируются на общих основаниях). Перед отправкой карточки каждый
неприкосновенный фрагмент маскируется последовательным плейсхолдером `{T1}`,
`{T2}`, … слева направо; модель возвращает те же `{Tn}` на тех же местах.
Особенность PWG, которой нет в `mw_ru`: `{%…%}` сам по себе — разделитель,
который нужно классифицировать, а не маскировать вслепую — немецкая глосса
разворачивается и переводится внутри скобок, латинская маскируется целиком.
