DharmaMitra demo passage — a G5-gated PWG card

Created: 02-07-2026 · Last updated: 02-07-2026

A single real dictionary card where our quality gate caught an AI translation that added content the source dictionary never states — the failure mode DharmaMitra's own questionnaire calls hallucinated content. It is the concrete illustration of this pipeline's one-line thesis: an updatable, verified dictionary.

The card: abhyastaṃgā (√gā "to set, go down")

Petersburger Wörterbuch (PWG), sub-entry keyed g_a~~h0_14_a_byastam; attested in the Śatapatha Brāhmaṇa.

1. German source — what the dictionary actually says

— abhyastam untergehen vor, bei, während einer Handlung u. s. w.: uddhṛtam abhyastam agāt — ŚAT. BR. 2,3,1,7. 4,4,6.

Plain English of the German gloss: "(indeclinable abhyastam) — to set / go down before, at, or during an action, etc." The German says nothing about what does the setting.

2. Raw AI translation — before the gate

Generated by Sonnet 4.6 (claude-sonnet-4-6) on the PWG→EN pilot path:

— abhyastam to set (of a heavenly body) before, during, in the course of an action u. s. w.: uddhṛtam abhyastam agāt …

The highlighted phrase (of a heavenly body) is invented. It has no counterpart in the German. It is a parenthetical in the house style of a different dictionary — Monier-Williams, which glosses this root "to go down, set (as a heavenly body)." The AI carried Monier-Williams in its translation memory and let that dictionary's editorial voice leak into a Petersburger card. The insidious part: the addition is contextually true — abhyastaṃgā really does describe the sun setting — so a generic chatbot user would accept it as helpful and never question it. But it is not what PWG says.

3. Corrected version — after the G5 human gate

— abhyastam to set before, during, in the course of an action, etc.: uddhṛtam abhyastam agāt …

The unsourced parenthetical is removed; the card faithfully mirrors the German and its provenance flips from ai_translated to approved.

Why this is an updatable verified dictionary, not just a chatbot

  1. Verified — an explicit human gate (G5). Every card carries a review_status (ai_translated → approved) and reviewer. Cards are judged against the German (here by Fable 5, claude-fable-5, session S7); this one was caught as failure class addition + mw-tm-contamination.
  2. Provenance on every card — model, generator, input hashes, timestamp — so any card traces to how it was produced, not an anonymous string.
  3. Auditable corrections. The fix is a change file in the csl-corrections audit trail, not an in-place overwrite — before/after and reason stay permanently inspectable.
  4. Updatable on a cadence. Validated corrections ship into the canonical csl-orig dictionary via a single monthly batched pull request — the dictionary improves continuously, unlike a frozen PDF.

No tool in the questionnaire's list offers both verification (a real gate + audit trail) and updatability (a live pipeline into a canonical dictionary). That combination is the differentiator — and this card is what it looks like when it fires.

Mapping to DharmaMitra's taxonomy: failure = hallucinated content (cross-dictionary translation-memory contamination); breakdown level = word/phrase; post-editing burden = one deletion.
