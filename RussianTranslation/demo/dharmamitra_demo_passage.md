# DharmaMitra demo passage — a G5-gated PWG card

_Created: 02-07-2026 · Last updated: 02-07-2026_

A one-page sample for the workshop **"Translation in the Age of AI: DharmaMitra
for Translator Workflows."** It shows a single real dictionary card where our
quality gate caught an AI translation that added content the source dictionary
never states — the exact failure mode DharmaMitra's own questionnaire calls
**hallucinated content**. It is the concrete illustration of the recorded
one-line thesis of this whole pipeline: an **updatable, verified dictionary**.

---

## The card: `abhyastaṃgā` (√gā "to set, go down")

Petersburger Wörterbuch (PWG), sub-entry keyed `g_a~~h0_14_a_byastam`; attested
in the Śatapatha Brāhmaṇa. Source line in
[csl-orig `v02/pwg/pwg.txt`](https://github.com/sanskrit-lexicon/csl-orig/blob/master/v02/pwg/pwg.txt).

### 1. German source (what the dictionary actually says)

> — *abhyastam* **untergehen vor, bei, während einer Handlung** u. s. w.:
> *uddhṛtam abhyastam agāt* — ŚAT. BR. 2,3,1,7. 4,4,6.

Plain English of the German gloss: *"(indeclinable **abhyastam**) — to set /
go down **before, at, or during an action**, etc."* The German says **nothing**
about what does the setting. It is a bare temporal-adverbial gloss.

### 2. Raw AI translation (before the gate)

Generated by Sonnet 4.6 (`claude-sonnet-4-6`) on the PWG→EN pilot path:

> — *abhyastam* to set **(of a heavenly body)** before, during, in the course
> of an action u. s. w.: *uddhṛtam abhyastam agāt* …

The bold phrase **"(of a heavenly body)"** is invented. It has **no counterpart
in the German**. It is a parenthetical in the characteristic house style of a
*different* dictionary — Monier-Williams, which glosses this root "to go down,
set (as a heavenly body)." The AI had Monier-Williams in its translation memory
and let that dictionary's editorial voice leak into a Petersburger card.

The insidious part: the addition is *contextually true* — *abhyastaṃgā* really
does describe the sun setting. A generic chatbot user would read "(of a heavenly
body)" as a helpful clarification and never question it. But it is **not what
PWG says**, and a dictionary's value is precisely that you can trust each card
to report *that* dictionary's evidence, not a plausible blend of several.

### 3. Corrected version (after the G5 human gate)

> — *abhyastam* to set before, during, in the course of an action, etc.:
> *uddhṛtam abhyastam agāt* …

The unsourced parenthetical is removed. The card now faithfully mirrors the
German, and its provenance flips from `ai_translated` to `approved`.

## Why this is an *updatable verified dictionary*, not just a chatbot

Four mechanisms, in order:

1. **Verified — an explicit human gate (G5).** Every AI-translated card carries
   a `review_status` (`ai_translated` → `approved`) and a `reviewer`. Cards are
   drawn into a stratified fidelity sample, judged against the German (here by
   Fable 5, `claude-fable-5`, session S7), and each flagged card is human-gated
   before promotion. This card was caught by the judge as failure class
   **addition + mw-tm-contamination** — see the
   [S7 Fable judge memo](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/FABLE_JUDGE_S7_2026-07-02.md).
2. **Provenance on every card.** Model, generator, input hashes, and timestamp
   ride with the translation, so any card can be traced to how it was produced
   and re-judged — not an anonymous black-box string.
3. **Auditable corrections.** The fix is not an in-place overwrite; it is
   expressed as a change file in the
   [csl-corrections](https://github.com/sanskrit-lexicon/csl-corrections) audit
   trail, so the before/after and its reason are permanently inspectable.
4. **Updatable on a cadence.** Accumulated, validated corrections are delivered
   into the canonical [csl-orig](https://github.com/sanskrit-lexicon/csl-orig)
   dictionary via a single **monthly batched pull request** — the dictionary
   improves continuously, unlike a frozen print-era PDF.

No tool in the workshop questionnaire's list offers this combination:
verification (a real gate + audit trail) **and** updatability (a live correction
pipeline into a canonical dictionary). That combination is the differentiator —
and this card is what it looks like when it fires.

**Mapping to DharmaMitra's taxonomy:** failure = *hallucinated content*
(specifically, cross-dictionary translation-memory contamination); breakdown
level = *word/phrase*; post-editing burden = *one deletion*.
