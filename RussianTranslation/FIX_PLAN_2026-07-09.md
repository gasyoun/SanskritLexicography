# PWG-RU Fix Plan - 2026-07-09

This is the actionable fix backlog for `RussianTranslation`, derived from past
launch failures and audits. It is intentionally operational: every item below
should either prevent silent wrong output, prevent wasted Max/API spend, or make
failures diagnosable.

Primary evidence:
- `LAUNCH_FUCKUPS.md`
- `PIPELINE_HISTORY.md`
- `PIPELINE_CAPABILITY_AUDIT_2026-07-08.md`
- `LANG_PARITY.md`

## P0 - must fix before next scale run

P0 must be complete before another scale Workflow/API launch.

### 1. Requeue/TM proof

**Prevents:** defect requeues silently serving the same failed card or fragment
from translation memory.

**Work:**
- Verify every defect/all requeue forces `--no-tm`.
- Verify failed fragment hashes are added to the TM denylist.
- Verify denylist reads fail loud on torn/invalid JSONL.

**Done when:**
- A gate-failed card cannot be re-served from card TM.
- A gate-failed fragment cannot be re-served from fragment TM.
- The requeue command/report records the exact cache-bypass decision.

**Minimum checks:**

```powershell
python src\pilot\window_selftest.py
python src\pilot\lang_parity_check.py
```

### 2. Gate truthfulness

**Prevents:** validators passing clean when child gates crash, prose parsing
changes, output is partial, or reports are unparseable.

**Work:**
- Confirm child auditors emit machine-readable verdicts.
- Confirm missing/malformed child verdicts fail loud.
- Confirm partial cards and fidelity drift survive into audit reports and store
  provenance.
- Confirm RU/EN audit parity where the behavior is language-neutral.

**Done when:**
- No child-gate crash can be interpreted as clean.
- No unparseable gate output can drop requeue candidates.
- Partial/fidelity flags are visible at audit and promotion boundaries.

**Minimum checks:**

```powershell
python src\pilot\window_selftest.py
python src\pilot\lang_parity_check.py
```

### 3. Launch-ledger enforcement

**Prevents:** failed launches closing without a durable incident record.

**Work:**
- Check every recent Workflow/API failure, retry, stale refusal, kill cascade,
  cost blow-up, or suspicious residual has a `LAUNCH_FUCKUPS.md` entry.
- Add closeout language to the operator checklist if any path can still skip the
  ledger.

**Done when:**
- Every failed launch since 2026-07-05 is classified.
- Each launch handoff can be checked by handoff id or date.

**Minimum checks:**

```powershell
python src\pilot\check_launch_ledger.py --since 2026-07-05
```

## P1 - fix before multi-account production

### 4. Concurrency and append safety

**Prevents:** shared-clone collisions, torn JSONL appends, status clobbering, and
unsafe multi-account production.

**Work:**
- Verify production runs use worktree-per-lane and disjoint root shards.
- Verify only one promoter writes merged store output.
- Verify append-only JSONL files use atomic append helpers.
- Verify singleton status/report files cannot be clobbered by tests or temp
  audits.

**Done when:**
- The direct production path is safe under the documented 3-account protocol.
- Concurrent writes either serialize, append atomically, or are isolated.

### 5. Artifact lifecycle

**Prevents:** watcher deletion, zombie overwrite, late writer clobbering, and
manual-save loss.

**Work:**
- Confirm generated outputs are staged outside the repo when watcher risk exists.
- Confirm landed outputs are sealed by hash.
- Confirm redispatch never reuses an output path until the prior writer is dead.

**Done when:**
- A late/zombie writer cannot overwrite a promoted or scored artifact.
- Every landed artifact has a post-land integrity check.

### 6. Lane-specific kill calibration

**Prevents:** applying dense-root timeout assumptions to singleton/no-PWG or
other lanes.

**Work:**
- Separate timeout assumptions for dense roots, no-PWG singles, presplit
  fragments, topups, and synthesis.
- Keep no-fallback singleton cards on the CEIL budget.
- Document when a recurrence means transient API instability vs true
  miscalibration.

**Done when:**
- Each launch lane has an explicit runtime budget model.
- A kill cascade produces a diagnosable classification, not guesswork.

## P2 - research and data-quality backfills

### 7. Per-sense evidence provenance

**Prevents:** losing the answer to "which source supported this sense?"

**Work:**
- Backfill per-sense evidence from deterministic joins where possible.
- Preserve missing evidence as explicit `evidence_unavailable`, not silence.

**Done when:**
- A promoted sense can expose supporting source/layer evidence or an explicit
  unavailable status.

### 8. Case-government backfill

**Prevents:** the existing `government` schema slot remaining empty despite known
PWG government markers.

**Work:**
- Use the existing government census as the source of truth.
- Backfill structured government data after the field-shape decision is settled.

**Done when:**
- Government markers are queryable per sense for the known marker population.

### 9. Evidence-status corpus gate audit

**Prevents:** absent evidence looking like true negative evidence.

**Work:**
- Verify corpus/evidence gates distinguish `ok`, `evidence_unavailable`,
  `db_absent`, `db_error`, and true uncovered terms.

**Done when:**
- Corpus gate reports cannot silently degrade to LLM-only without an explicit
  status marker.

### 10. Windows, encoding, and path hygiene

**Prevents:** CRLF key failures, BOM bugs, cp1252 subprocess decoding, safe-name
collisions, and key-echo drops.

**Work:**
- Sweep key-list readers for CRLF handling.
- Sweep subprocess readers for `encoding='utf-8'`.
- Verify BOM handling on source reads/writes.
- Verify exact-case filename handling and safe-name/key mapping.

**Done when:**
- Known Windows/path/encoding traps have deterministic tests or explicit guards.
