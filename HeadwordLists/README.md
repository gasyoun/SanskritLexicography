# HeadwordLists

Exported and derived **headword lists** for Cologne Digital Sanskrit Lexicon
(CDSL) dictionaries, plus cross-dictionary comparison files. Each list is one
headword per line, sorted, used for coverage analysis, deduplication, and
joining dictionaries against one another.

Encoding is **SLP1** in the `key1`/`key2` files (e.g. `aMSa`, `a/MSa`); the one
exception is [21562-huet-velthius.txt](then-2014/21562-huet-velthius.txt), which is in
**Velthuis** transliteration (e.g. `a.mza` = aṃśa). All files are UTF-8.

## Naming scheme

```
{DICT}-unique-{key1|key2}-{N}.txt
```

- `{DICT}` — CDSL dictionary code (see table below).
- `key1` / `key2` — which headword key (see next section).
- `{N}` — **entry count**. Note: `wc -l` usually reports `N − 1` because the
  last line has no trailing newline; the filename count is the true number of
  entries.

Variant patterns:

| Pattern | Meaning |
|---|---|
| `{DICT}-fehlerhaft-{N}.txt` | German "erroneous" — entries flagged as problematic. **These hold full XML records, not bare headwords** (see [PWG-fehlerhaft-1661.txt](then-2014/PWG-fehlerhaft-1661.txt), [PWK-fehlerhaft-2227.txt](then-2014/PWK-fehlerhaft-2227.txt)). |
| `SCH-accents-IAST-{N}.txt` | Accented headwords in IAST rather than SLP1 ([SCH-accents-IAST-20247.txt](then-2014/SCH-accents-IAST-20247.txt)). |
| `{N}-huet-velthius.txt` | Count as a **prefix**; Velthuis transliteration ([21562-huet-velthius.txt](then-2014/21562-huet-velthius.txt)). **Non-Cologne**: the INRIA "Sanskrit Heritage" reader stem list (Huet). Coverage/corpus analysis: [Huet-INRIA-Wordlist-vs-Cologne.md](Huet-INRIA-Wordlist-vs-Cologne.md). |
| `mw-apte-mcdonell-hk.txt` | Aggregate cross-dictionary list in **Harvard-Kyoto**, MW ∪ Apte ∪ Macdonell (~202k lines) ([mw-apte-mcdonell-hk.txt](then-2014/mw-apte-mcdonell-hk.txt)). |
| `sanhw1.xlsx` | 41 MB binary spreadsheet — **do not open in an editor**; load with a library. |

## key1 vs key2 — choose deliberately

- **key1** — normalized computational key. May not match any printed form; built
  for machine comparison. Use for **matching, dedup, joins**.
- **key2** — closer to the printed source; retains markers like `-`, `--`, `/`
  (accent), `°` (e.g. `a/MSa`, `a--kAra`, `aMSa°prakalpanA`). Use for
  **editorial review, citation, checking digitized text against the scan**.

Not every dictionary ships both: some have only `key2` here (e.g. BHS, BUR, CAE,
CCS, MD, SCH).

## then-2014 vs now-2026 — these snapshots have drifted from current csl-orig

The lists were **extracted in 2014** (first commit 2014-10-05, "Cologne headwords") and
now live frozen in [`then-2014/`](then-2014/); their filename count `N` is the *then*
count. [`NOW_VS_THEN.md`](NOW_VS_THEN.md) (regenerated by
[`headword_diff.py`](headword_diff.py)) compares each against the current (2026) csl-orig,
with `overlap`, `growth %`, and a **TOTAL** row. The **key1** (SLP1) lists are directly
comparable — e.g. **AP 36,030 → 88,867 (+146.6 %)**, **PWK 131,918 → 151,349 (+14.7 %)**,
**MW 193,978 → 194,084**, with PWG/GRA/SKD/VCP/VEI nearly flat; across the 9 comparable
lists **605,813 → 733,617 (+21.1 %)**. The **key2** snapshots are in the *legacy Cologne
numeric transliteration* (`am2s4a` = aṃśa) whereas csl-orig is now SLP1, so their raw diff
is a format migration, not a headword change. PD is not in csl-orig.

The **current** regeneration of **both keys** lives in [`now-2026/`](now-2026/) (the
`then-2014/` files are kept frozen for comparison); see
[`now-2026/README.md`](now-2026/README.md). **key1** (machine key) is directly
set-comparable to the 2014 file; **key2** is re-extracted as clean **SLP1** — the
print/citation form (keeps `/` accent, `-`/`—`, `(...)`) — since the 2014 key2 was legacy
numeric, key2 then-vs-now is a format migration (not line-diffable) but the now-2026 key2
is directly usable. Compare by set, not raw line-diff (now-2026/ uses Sanskrit collation).

**Print-readiness:** [`PRINT_READINESS.md`](PRINT_READINESS.md) consolidates whether the
headword lists can go to a printed edition — MW/PWG are the stable print-ready spine; the
remaining gates (typo-filing, coverage, accents, scope) are human/editorial.

**Alternate & feminine headwords:** [`ALTERNATE_HEADWORDS.md`](ALTERNATE_HEADWORDS.md)
(PRINT_READINESS item F) — machine-generated candidate worklists (feminine↔masculine pairs,
variant spellings, alternate forms) for the editor; data in [`f_candidates/`](f_candidates/),
generated by [`alternate_headwords.py`](alternate_headwords.py).

**Typo queue (item A):** [`A_TYPO_QUEUE.md`](A_TYPO_QUEUE.md) — the MW 4 + PWG 12 body-confirmed
fileable typos for the print spine (IAST + error type + entry-body evidence), assembled from
SanskritSpellCheck by [`assemble_typo_queue.py`](assemble_typo_queue.py); verify on scan, then file.

**Coverage additions (item B):** [`COVERAGE_ADDITIONS.md`](COVERAGE_ADDITIONS.md) — DCS-corpus
lemmas absent from all CDSL dicts, ranked by frequency band (21,759 total; priority = 409
band-3 nominal; the rest low-band or lemmatisation artifacts). Built by
[`coverage_additions.py`](coverage_additions.py).

**Cross-dict union (the print target):** [`union/UNION.md`](union/UNION.md) — a single merged
headword index across all **15** csl-orig dicts (**323,425** headwords with per-dict provenance
+ gender), feminines folded under the masculine (`mf(ā/ī)`; 237 `-inī` auto-folded, 3,995 `-ā/-ī`
candidates ranked high/low for the editor); built by [`build_union.py`](build_union.py).

## Note on duplicates / versions

Two MW `key2` files are present — [MW-unique-key2-198220.txt](then-2014/MW-unique-key2-198220.txt)

## Encoding caveat — BOM is inconsistent

Some files carry a UTF-8 BOM and some do not (e.g.
[MW-unique-key1-193978.txt](then-2014/MW-unique-key1-193978.txt) and the
huet-velthius file **have** a BOM; [MW-unique-key2-198220.txt](then-2014/MW-unique-key2-198220.txt)
does **not**). Check `head -c 3 file | xxd` before transforming, and preserve the
file's existing BOM state on write. The org-level "no BOM" rule does **not** hold
in this directory.

## Subdirectory: `Catalan-Pujol/`

An **external** Sanskrit headword spine and its coverage analysis against CDSL —
not a CDSL export. Source: the *Diccionari Sànscrit–Català* (Òscar Pujol,
Enciclopèdia Catalana, 2005, the first Sanskrit→Catalan dictionary). The lemma
list is mirrored from the `sanskrit-lexicon/CORRECTIONS` repo.

| File | What it is |
|---|---|
| [Catalan-Pujol/61267-Sanskrit-Catalan-Words-List.txt](Catalan-Pujol/61267-Sanskrit-Catalan-Words-List.txt) | The raw lemma spine — 61,266 lines, **accented IAST with `√`-roots, Vedic udātta, and Pujol's compound-segmenting hyphens**, UTF-8 **with BOM**. |
| [Catalan-Pujol/Sanskrit-Catalan-Wordlist-vs-Cologne.md](Catalan-Pujol/Sanskrit-Catalan-Wordlist-vs-Cologne.md) | The analysis. **Dictionary axis** (§2–4): the list is ≈ a subset of Monier-Williams (**MW alone covers 88.5 %**; 91 % with all 15 dicts). **Corpus axis** (§5, vs DCS): only **46.4 %** is attested in the DCS corpus — **44.9 % is dictionary-listed but corpus-unattested** ("dark matter"); the 0.3 % corpus-attested-with-no-CDSL-entry corner is **triaged** (≈55 lemmatisation/morphology convention vs a genuine residue of corpus-attested rare lexemes — plant/animal names — absent from all 43 dicts). **§6** documents **Pujol's 11 headword conventions** (`√`-roots, preverb segmentation, udātta, compound hyphens, stem/feminine/suffix forms, bīja syllables, BOM/`ś` encoding); **§7** = accent comparison vs Cologne (notation differs, but **~97 %** put the udātta on the same vowel; GRA 96.9 %, MW 97.1 %). Also covers the two transcoding traps. Full 177-lemma list in [DCS-attested-no-CDSL.md](Catalan-Pujol/DCS-attested-no-CDSL.md). |
| [Catalan-Pujol/Catalan-uncovered-by-CDSL.txt](Catalan-Pujol/Catalan-uncovered-by-CDSL.txt) + [Catalan-uncovered/](Catalan-Pujol/Catalan-uncovered/) | The ~4.7k lemmas no CDSL dictionary covers, bucketed (simple / compound / root / prefixed-root / suspect-char). |
| `coverage_by_dict.py`, `match_rate.py`, `make_uncovered_lists.py`, `coverage_vs_dcs.py`, `accent_compare.py` | The analysis scripts (repo-portable; IAST→SLP1 via [`sanskrit-util`](https://github.com/sanskrit-lexicon/sanskrit-util)). `coverage_vs_dcs.py` cross-tabs dictionary × DCS-corpus attestation (`VisualDCS/dcs_lemma_summary.json`) and emits `DCS-attested-no-CDSL.md`; `accent_compare.py` checks Pujol's udātta placement against Cologne `<k2>`. |

> ⚠️ Unlike the `key1`/`key2` files here, this list is **accented IAST**, not SLP1,
> and carries an editorial compound-segmentation layer absent from any CDSL key —
> normalize (strip `√`, accents, hyphens; IAST→SLP1) before matching.

## Entry points

1. `*-unique-key1-*.txt` — normalized matching, dedup, joins.
2. `*-unique-key2-*.txt` — print-form display, citation, scan checking.
3. [mw-apte-mcdonell-hk.txt](then-2014/mw-apte-mcdonell-hk.txt) — ready-made comparison
   across three major dictionary traditions.
4. [Catalan-Pujol/](Catalan-Pujol/) — external Sanskrit–Catalan wordlist + its
   CDSL-coverage analysis (see subsection above).
5. [Huet-INRIA-Wordlist-vs-Cologne.md](Huet-INRIA-Wordlist-vs-Cologne.md) — the
   non-Cologne INRIA Heritage stem list, same coverage/corpus analysis (a control
   for the Catalan study: 86.2 % CDSL-covered, **60.0 % corpus-attested** vs Pujol's
   46.4 %).
